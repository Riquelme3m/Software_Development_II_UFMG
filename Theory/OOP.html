What is Object Oriented Programming?

It's a programming paradigm where the development is driven by objects which communicate to each other where those objects are classes with attributes and methods.

Why OOP?

  • Organization
  • Readability
  • Maintainability
  • Reusability
  • Scalability
  • Collaboration

What's class?

Classes are a user-defined type or in another words blueprint of an object which contains attributes and methods .

What's an object?

An object is the instance of a class , which contains the attributes and methods implemented in that class and perform operations.

What are the main features of OOP?

  • Inheritance
  • Polymorphism
  • Abstraction
  • Encapsulation

What is encapsulation?

Encapsulation is the process of wraping attributes and methods within a class restricting direct access to some details.

What is abstraction?

Abstraction is the process of hiding complex implementation details and showing only the main features of an object.It allows the users to interact with the objects without needing to understand internal details.

What is polymorphism?

Polymorphism is the capacity of a function to behave differently depending on the context.

What are the two types of polymorphism?

  • Compile time polymorphism (static polymorphism) which is achieved by method overloading or operator overloading.
  • Run time polymorphism (dynamic polymorphism) which is achieved by method overriding.

What is inheritance?

Inheritance is the process of a class where this class inherits either methods or attributes from another class and reuse them in a relationship parent-child.

What is the difference between a structure and a class in C++

In a struct everything is public either attributes or methods . In a class , by default everything is private unless you set otherwise. We usually use struct for implementing data structures such as nodes and so on .

What is a constructor?

A constructor is a function that initializes an object . In C++ it's important to understand the difference between the default constructor that you might set and the constructor with the member initializer list

In the default constructor , the attributes are initialized twice ( one for the default initialization ) and one for the one you set , but when you use lists there is not this default initialization which is more efficient.

A constructor is called every time a new object is created

What is a destructor?

A destructor is a method which is called when either an object goes out of scope or when the keyword delete is called.

In the example virus , we see a kind of recursion calling the destructors in a chain effect.

What are access modifiers/operator modifiers ?

They are keywords that we use to restric access to methods and attributes in a class . In C++ we have three :

  • Private : Only available within the class
  • Public : Available anywhere
  • Protected: Available within the class and for its children

What is static method and attributes ?

When you create a new object , a new memory address is assigned to that object . However, when you create a static method or attribute , this one occupies only one address in memory . Therefore , it means that we can access this methods or attribute from any object created or using the class name itself removing the need to create a new object. Example : static int objectCounter which is incremented by one every time a new object is created

How much memory a class occupy?

A class does not occupy any space in memory since it's only the template/blueprint items are made. Only objects occupy memory.
