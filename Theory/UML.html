What is UML?

Stands for Unified Modeling Language and it's basically a graphical notation largely used in software engineering.

How access modifiers are represented in UML?

  • +:public
  • -:private
  • #:protected

What is Association

When two classes can exist independently and there is no ownership

Example : Teacher - Student

Aggregation

When there is a weak ownership but both classes can exist independently

Example : Library - book

Composition

When there is a strong ownership and the part does not exist withour a whole

Example : Car - Engine

Dependency

When there is no ownership and the classes can exist independently

Example : Doctor - medication ( parameter of a function)

Inheritance

Parent child ownership

Both classes can exist independently

What are user stories ?

It's a way to write requirements for a software project which contains the descriptions and the acceptance criteria

Me as a user want to access my weekly agenda for me to be aware of my tasks

Acceptance criteria :

  • Visualize my weekly agenda
  • Visualize my daily tasks
